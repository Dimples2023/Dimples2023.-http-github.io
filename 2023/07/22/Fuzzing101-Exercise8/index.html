Fuzzing101-Exercise8

分类：
标签： 实验报告

日期：2023-07-22 21:19:53

安装QEMU

下载并安装QEMU

sudo apt install ninja-build libc6-dev-i386
cd /home/lbj/AFLplusplus/qemu_mode/
CPU_TARGET=i386 ./build_qemu_support.sh
sudo cp '/home/lbj/AFLplusplus/afl-qemu-trace' /usr/local/bin/

检查是否安装完成

afl-qemu-trace --help

出现以下画面则成功安装

安装 Adobe Reader

安装依赖项

sudo apt-get install libxml2:i386

下载并解压缩：AdbeRdr9.5.1-1_i386linux_enu.deb

![微信图片_20230722165845](C:/Users/27783/Desktop/微信图片_20230722165845.png)wget ftp://ftp.adobe.com/pub/adobe/reader/unix/9.x/9.5.1/enu/AdbeRdr9.5.1-1_i386linux_enu.deb

安装AdbeRdr9.5.1-1_i386linux_enu.deb

sudo dpkg -i AdbeRdr9.5.1-1_i386linux_enu.deb

运行以下命令

./opt/Adobe/Reader9/bin/acroread

接受许可协议后，应看到 Adobe Reader 界面

创建种子语料库

下载并解压缩语料库

wget https://corpora.tika.apache.org/base/packaged/pdfs/archive/pdfs_202002/libre_office.zip
unzip libre_office.zip -d extracted

仅复制小于 2 kB 的文件，以加快模糊测试过程

mkdir -p $HOME/fuzzing_adobe/afl_in
find ./extracted -type f -size -2k \
    -exec cp {} $HOME/fuzzing_adobe/afl_in \;

Fuzz

方法一

使用 -Q 参数运行 afl-fuzz

ACRO_INSTALL_DIR=/opt/Adobe/Reader9/Reader ACRO_CONFIG=intellinux LD_LIBRARY_PATH=$LD_LIBRARY_PATH:'/opt/Adobe/Reader9/Reader/intellinux/lib' afl-fuzz -Q -i ./afl_in/ -o ./afl_out/ -t 2000 -- /opt/Adobe/Reader9/Reader/intellinux/bin/acroread -toPostScript @@

运行 afl-fuzz 时需要小心，因为它是一个 shell 脚本，真正的二进制路径如下：/opt/Adobe/Reader9/bin/acroread``/opt/Adobe/Reader9/Reader/intellinux/bin/acroread

可以看到AFL++运行

但模糊测试的速度比较慢，因此接下来使用持久模式

持久模式

与实验6中，通过插入 AFL_LOOP 的方法启用持久模式不同，在本实验中，由于无法访问源码，所以使用 AFL_QEMU_PERSISTENT_ADDR 来指定持久循环的开始，使用持久模式。建议将此地址设置为函数的开头

安装 valgrind 和 kcachegrind

sudo apt-get install valgrind
sudo apt-get install kcachegrind

生成一个报告

ACRO_INSTALL_DIR=/opt/Adobe/Reader9/Reader ACRO_CONFIG=intellinux LD_LIBRARY_PATH=$LD_LIBRARY_PATH:'/opt/Adobe/Reader9/Reader/intellinux/lib' valgrind --tool=callgrind /opt/Adobe/Reader9/Reader/intellinux/bin/acroread -toPostScript [samplePDF]

其中samplePDF是 PDF 示例的路径。将在当前的文件夹中生成一个输出文件callgrind.out。现在，我们可以使用以下命令可视化运行 kachegrind 的此报表

kcachegrind

将显示以下画面

查看 kcachegrind 中的字段，以识别仅执行 1 次的函数，并尝试在 afl-fuzz 中实现超过 90% 的稳定性

经过多次尝试，最终选择地址为0x0850ca28的函数

进行fuzz

AFL_QEMU_PERSISTENT_ADDR=0x085478AC AFL_QEMU_PERSISTENT_GPR=1 ACRO_INSTALL_DIR=/opt/Adobe/Reader9/Reader ACRO_CONFIG=intellinux LD_LIBRARY_PATH=$LD_LIBRARY_PATH:'/opt/Adobe/Reader9/Reader/intellinux/lib' afl-fuzz -Q -i /home/lbj/fuzzing_adobe/afl_in/ -o ./afl_out/ -t 2000 -- /opt/Adobe/Reader9/Reader/intellinux/bin/acroread -toPostScript @@

可以发现运行速度显著提升

检查

AFL_USE_QASAN=1 ACRO_INSTALL_DIR=/opt/Adobe/Reader9/Reader ACRO_CONFIG=intellinux LD_LIBRARY_PATH=$LD_LIBRARY_PATH:'/opt/Adobe/Reader9/Reader/intellinux/lib' /usr/local/bin/afl-qemu-trace -- /opt/Adobe/Reader9/Reader/intellinux/bin/acroread -toPostScript [crashFilePath]

crashFilePath为崩溃文件的路径

明明路径下有libqasan.so

但仍然显示无法找到该文件，尚不知原因

总结

  • 学习了如何使用 AFL++ 的 QEMU 模式模糊闭源应用程序
  • 学习了如何在 QEMU 模式下启用持久模式
  • 学习了如何使用QASAN
